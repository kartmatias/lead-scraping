# Lead Scraping — Project Context

**Path:** /mnt/operacional/projects-2026/lead-scraping  
**Stack:** Laravel (PHP), SQLite (dev) / PostgreSQL 16 (prod), Redis (queues & cache) / Database (dev queue), Guzzle HTTP Client

## Architecture

Queue-based async pipeline:
1. `POST /api/scrape-requests` → creates `ScrapeRequest` (status: `pending`)
2. `RunApifyActor` job → calls Apify API to start actor run, stores `apify_run_id` + `apify_dataset_id`, sets status to `running`, and dispatches `ProcessScrapeRequest`
3. `ProcessScrapeRequest` job → polls Apify run status, fetches dataset items when `SUCCEEDED` (uses `$this->release(30)` for polling), updates scrape request counters, and imports leads
4. `Lead::upsertFromScrapeData()` → upserts leads by `(source_type, source_id)` unique key
5. `LinkLeadsJob` job → matches imported leads to `CanonicalEntity` records and links them using the `lead_links` pivot table

## Canonical Linkage Logic (`LinkLeadsJob`)

Matches newly imported leads to `CanonicalEntity` records using the following priority:
1. **CNPJ** (exact)
2. **Phone** (normalized to E.164 `+55DDNNNNNNNNN`)
3. **Email** (exact)
4. **Website** (normalized: strip protocol/www and trailing slashes)
5. **Name fuzzy** (`similar_text` match ≥ 85% after ASCII normalization)
6. If no match, a new `CanonicalEntity` is created.

Updates matched entities using `CanonicalEntity::mergeFromLead()` to fill in empty/null fields with newly scraped data.

## Apify Actors

Actor IDs in `ApifyActorRegistry` contain `/` (e.g., `compass/crawler-google-places`), but are internally converted to `~` (e.g., `compass~crawler-google-places`) before querying the Apify API in `ApifyService`.

| Source      | Actor ID                            | Input Example / Notes |
|-------------|-------------------------------------|-----------------------|
| google_maps | compass/crawler-google-places       | `{"searchStringsArray":["restaurants in São Paulo"],"maxItems":10}` |
| instagram   | apify/instagram-scraper             | `{"directUrls":["https://www.instagram.com/user/"],"resultsLimit":10}` |
| linkedin    | dev_fusion/linkedin-profile-scraper | Decision maker profile data. Requires full-permission approval. |
| cnpj        | parseforge/brazil-cnpj-scraper      | `{"cnpjs":["00000000000191"]}` — *Note: The field name is plural (`cnpjs`).* |

## Key Files

- `app/Services/ApifyService.php` — Guzzle wrapper: `startActorRun`, `getRunStatus`, `getDatasetItems`, `getAllDatasetItems`, `cancelRun`, `getActorDetails`
- `app/Jobs/RunApifyActor.php` — triggers actor runs; actor_id resolved via match on `scrapeRequest->source`
- `app/Jobs/ProcessScrapeRequest.php` — polls + persists leads (tries=3, backoff=30)
- `app/Jobs/LinkLeadsJob.php` — matches leads to canonical entities
- `app/Models/Lead.php` — `upsertFromScrapeData()`, belongsTo `ScrapeRequest`, hasOne `LeadLink`
- `app/Models/ScrapeRequest.php` — `startScrape()`, `complete()`, `fail()`, `cancel()`, `isPending()`, `isRunning()`
- `app/Models/CanonicalEntity.php` — Enriched entity merged from multiple leads; hasMany `LeadLink`
- `app/Models/LeadLink.php` — Pivot: lead → canonical entity with `match_method` + `match_score`
- `app/Providers/ApifyActorRegistry.php` — actor list; uses `~` separator internally
- `app/Providers/ApifyActorValidationServiceProvider.php` — validates all actors at boot; skipped if `APIFY_VALIDATION_DISABLED=true` or testing env
- `routes/api.php` — `/api/scrape-requests` (CRUD + status + cancel), `/api/leads` (index, stats, export, show, destroy)

## DB Schema

- **scrape_requests**: `id`, `source` (enum), `status` (enum), `filters` (JSON), `apify_run_id`, `apify_dataset_id`, `total_leads`, `completed_leads`, `failed_leads`, `started_at`, `completed_at`, `error_message`, `timestamps`
- **leads**: `id`, `scrape_request_id` (FK cascade), `source_type`, `source_id`, `name`, `email`, `phone`, `company`, `position`, `address`, `cnpj`, `website`, `instagram`, `linkedin`, `facebook`, `raw_data` (JSON), `timestamps`  
  *UNIQUE:* `(source_type, source_id)`
- **canonical_entities**: `id`, `name`, `cnpj`, `phone`, `email`, `website`, `address`, `instagram_url`, `linkedin_url`, `timestamps`  
  *UNIQUE:* `cnpj`
- **lead_links**: `id`, `canonical_entity_id` (FK cascade), `lead_id` (FK cascade), `match_method` (enum), `match_score`, `timestamps`  
  *UNIQUE:* `lead_id`
